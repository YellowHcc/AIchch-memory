**Slogan**："找出内鬼，保卫财报" / "谁是那个对不上账的人？"

## 三、核心机制

完成任务 → 赚「特工积分」→ 积分换抽奖券

| 任务类型 | 人数 | 时长 | 奖励 |
|---------|-----|------|-----|
| 单人认证 | 1人 | 2分钟 | 1分 |
| 多人缉凶 | 3-5人 | 5-8分钟 | 2-3分 |

**隐藏规则**：
- 完成多人任务+正确找出"内鬼" → 额外1分
- 集齐5分 → 「特工勋章」+ 暴富券

## 六、单人任务（了解部门）

### 任务①：「特工档案」—— 认识部门成员

| 项目 | 内容 |
|-----|------|
| **玩法** | 看5张部门成员照片，选择对应的岗位（销售/研发/交付/领导） |
| **形式** | 翻牌式——翻照片猜岗位，猜对3人过关 |
| **变体** | 人多：看3张猜对2人；人少：看5张猜对4人 |
| **道具** | 照片卡片（20人，每人1-2张）+ 岗位选项卡 |
| **奖励** | 1分 + 「特工证」卡片 |
| **了解点** | 部门有哪些人、组织架构 |

### 任务②：「产品密码」—— 了解产品能力

| 项目 | 内容 |
|-----|------|
| **玩法** | 场景配对——左边业务痛点，右边产品名称，连线配对 |
| **配对示例** |  |
| "月底三方数据对不上" | ↔ | 三方对账 |
| "要算清楚每笔收入该归谁" | ↔ | 资金清分 |
| "发票开错了要重新来" | ↔ | 发票管理 |
| "原始凭证要推给ERP" | ↔ | 凭证生成与推送 |
| "各种数据源格式乱七八糟" | ↔ | 数据接入与清洗 |
| **变体** | 人多：4选2配对；人少：8选4配对 |
| **道具** | 磁性卡片（可吸在白板上） |
| **奖励** | 1分 |
| **了解点** | 主要产品/能力 |

### 任务③：「流程解密」—— 了解工作模式

| 项目 | 内容 |
|-----|------|
| **玩法** | 看一段虚拟"业务事故"描述，选择是哪个产品环节出了问题 |
| **题目示例** | "销售说客户付款了，但财务账上没显示到账"——是哪里出问题？（答案：数据接入/三方对账环节） |
| **变体** | 人多：选择题3选1；人少：开放式回答 |
| **奖励** | 1分 |
| **了解点** | 产品应用场景、工作模式 |

## 七、多人任务（3-5人狼人杀类）

### 游戏A：「谁是问题单据」（主推）

**人数**：3-5人（满3人开局，最多5人）  
**时长**：5-8分钟  
**场地**：站立围圈

#### 角色配置

| 人数 | 好人（正常单据） | 狼人（问题单据） |
|-----|-----------------|-----------------|
| 3人 | 2人 | 1人 |
| 4人 | 3人 | 1人 |
| 5人 | 3人 | 2人 |

#### 单据卡设计

**好人卡示例**：
- "单据A：金额10000元，发票专票，审批链完整，三方对账一致"
- "单据B：金额5000元，电子发票，已关联合同，数据已清洗"

**狼人卡示例**（隐藏问题）：
- "单据X：金额10000元【实际应为8000】，发票专票，审批链完整"
- "单据Y：金额5000元，发票普票【应为专票】，审批链缺一级"
- "单据Z：金额3000元，发票合规，【三方对账金额不一致】"

#### 玩法流程

1. **发身份**：每人抽1张单据卡（保密，只看自己的）
2. **描述环节**（3轮）：
   - 轮流描述自己的单据（只能说属性，不能报具体金额）
   - 示例："我的单据三方对账一致"、"我的发票类型正确"
3. **讨论投票**（1-2轮）：讨论谁可能是"问题单据"，投票找出狼人
4. **揭晓**：
   - 好人赢：找出所有狼人 → 好人各得2分，找对人的额外+1分
   - 狼人赢：隐藏到最后 → 狼人各得3分

#### 摊位控制

- **排队>5组**：控制每局5分钟，限时发言（每人30秒）
- **排队<3组**：可延长讨论时间，增加互动

---

### 游戏B：「系统故障排查」

**人数**：3-5人  
**时长**：5-8分钟

#### 背景
财务流程出bug了！数据从"接入→清洗→对账→确认→凭证"某个环节断了。

#### 角色配置

每人代表一个节点：
- 数据接入员（好人/狼人）
- 数据清洗员（好人/狼人）
- 三方对账员（好人/狼人）
- 收入确认员（好人/狼人）
- 凭证生成员（好人/狼人）

狼人知道自己环节"数据异常"，但要假装正常。

#### 玩法

1. 发身份卡（写明自己的节点状态：正常/异常）
2. 轮流描述："我这一环节输入是XX，输出是XX"
3. 好人要找出哪个节点"数据对不上"
4. 投票找出故障节点

#### 示例发言
- 好人："我接收到的数据是已清洗格式，输出给对账环节"
- 狼人（故障节点）："我...接收到的数据...应该是正常的..."（心虚）

---

### 游戏C：「预算攻防战」

**人数**：4人（2v2对抗）

#### 角色
- 财务BP组（2人）：知道真实预算，要找出虚报
- 业务方组（2人）：其中1人虚报预算，1人正常申请

#### 玩法

1. 给出一个项目背景："市场部Q2要办一场线下活动"
2. 业务方组各自报预算金额和理由
3. 财务BP组提问质疑（如"为什么场地费这么贵？"）
4. 财务BP投票：谁虚报了预算？

#### 奖励
- 财务BP找对 → 各得2分
- 业务方隐藏成功 → 各得2分
