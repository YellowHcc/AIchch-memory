# 银联数据比对工具

## 功能概述

自动比对银联结算账单和银行流水，解决日常财务对账需求：

1. **数据预处理** - 自动识别日期格式、清洗门店名称、转换金额
2. **智能匹配** - 门店名称模糊匹配 + **按门店+日期分组汇总后金额比对**
3. **终端号反查** - 支持按终端号分组匹配，处理多终端号门店场景
4. **日期区间匹配** - 支持跨天匹配（如 0312-0314 区间）
5. **差异报告** - 输出未匹配流水组和结算单，便于核查

## 版本说明

| 版本 | 功能特点 | 适用场景 |
|------|----------|----------|
| **V1** | 逐条流水匹配结算单 | 一笔流水对应一笔结算 |
| **V2** | 按门店+日期分组汇总后比对 | 多笔流水对应一笔结算 |
| **V4** | 增加金额反查，使用**清分金额**比对 | 有优惠场景 |
| **V5** | 增加**终端号反查** | 多终端号门店 |
| **V6**（推荐） | 优化终端号反查：**支持日期区间** + **同一终端号分多笔打款组合匹配** | 复杂对账场景 |

```bash

```

### V6 核心特性

**1. 四层金额反查体系**

| 层级 | 说明 | 示例 |
|------|------|------|
| 第一层 | 门店匹配+金额反查 | 宸海记济南：455.89+552.83=1008.72 |
| 第二层 | 纯金额反查 | 食满天天水：151.30=151.30 |
| 第三层 | 日期汇总反查 | 未莱嘉晟珠海：4512.39+1919.45=6431.84 |
| **第四层** | **终端号反查** | 同一终端号多笔打款组合匹配 |

**2. 终端号反查（V5/V6 新增）**

针对多终端号门店场景：
- 从银行流水摘要中提取终端号
- 按终端号分组汇总匹配无门店流水
- **V6 优化**：支持日期区间（如 0312-0314），支持同一终端号分多笔打款的组合匹配

**示例场景**：
```
结算单：朝阳门店 0312 终端号001 净额 5000元
银行流水：
  - 银联入账：朝阳门店品牌一店001（0312） 2000元
  - 银联入账：朝阳门店品牌一店001（0312） 1500元  
  - 银联入账：朝阳门店品牌一店001（0312） 1500元
              ↓ 终端号001汇总
              5000元 ✓ 匹配成功
```

**3. 营业日期区间匹配（V6 新增）**

支持跨天对账场景：
```
摘要示例："银联入账：朝阳门店品牌一店（0312-0314）"
表示该笔结算包含 3月12日 至 3月14日 的汇总数据
```

**4. 输出报告结构**

生成的Excel报告包含11+个工作表：

| Sheet | 内容 |
|-------|------|
| 比对结果 | 所有流水组的匹配情况 |
| 已匹配 | 完全匹配的记录 |
| 金额反查匹配 | 门店匹配+金额反查补足的记录 |
| 纯金额反查匹配 | 无门店名但金额匹配的记录 |
| 日期汇总反查匹配 | 按日期汇总无门店流水匹配的记录 |
| **终端号反查匹配** | 按终端号分组匹配的记录（V5+） |
| 金额不符 | 到账与结算金额不一致的记录 |
| 无法匹配 | 未找到对应结算单的记录 |
| 未匹配结算单 | 有结算单但无对应流水的记录 |
| 未匹配无门店流水 | 无门店名也未匹配的记录 |
| 所有流水明细 | 全部流水原始数据 |
| 汇总 | 统计信息 |

## V6 匹配逻辑详解

**步骤1：门店映射**  
使用模糊匹配建立流水门店→结算单门店的映射（相似度≥50%）

**步骤2：终端号提取**  
从银行流水摘要中提取终端号：
- 模式：`品牌X店XXX` 或 `店XXX` 中的数字部分
- 示例：`"品牌一店001"` → 终端号 `001`

**步骤3：日期区间解析**  
从摘要中提取日期范围：
- `0312-0314` → 3月12日至3月14日
- `0312` → 3月12日（单日）

**步骤4：分组汇总比对**  
按（门店+终端号+日期）分组汇总后比对清分金额

**步骤5：四层金额反查**  
1. **门店匹配+金额反查** - 用无门店流水补足差额
2. **纯金额反查** - 无门店名但金额匹配
3. **日期汇总反查** - 按日期汇总匹配
4. **终端号反查** - 按终端号匹配（V6优化支持区间和组合）

## 输入数据要求

### 结算账单示例格式

| 门店 | 日期 | 交易金额 | 手续费 | 清分金额 | 终端号 |
|------|------|----------|--------|----------|--------|
| 朝阳门店 | 2024-03-01 | 260.00 | 1.16 | 251.04 | 001 |
| 海淀分店 | 2024-03-01 | 152.00 | 0.70 | 151.30 | 002 |

**必需字段**：POS门店名称、结算日期、清分金额（或清算金额）、终端号（可选）

### 银行流水示例格式

| 日期 | 收入金额 | 摘要 |
|------|----------|------|
| 2024-03-01 | 251.04 | 银联入账：朝阳门店品牌一店001（0301） |
| 2024-03-01 | 151.30 | 银联入账：海淀分店品牌二店002（0301） |

**摘要格式**：建议包含门店名、品牌店标识、终端号、日期区间，如：
```
银联入账：朝阳门店品牌一店001（0312-0314）
```

## 常见问题

**Q: V6 和 V4 有什么区别？**  
A: V6 在 V4 基础上增加了：
- **终端号反查** - 按终端号分组匹配
- **日期区间支持** - 跨天匹配（0312-0314）
- **组合匹配** - 同一终端号多笔打款组合匹配

**Q: "终端号反查"是什么意思？**  
A: 对于有多个终端号的门店，系统会：
1. 从摘要中提取终端号（如 `品牌一店001` → `001`）
2. 按终端号分组汇总流水
3. 匹配同一终端号的结算单

**Q: 日期区间怎么理解？**  
A: 支持两种格式：
- 单日：`0312` 表示3月12日
- 区间：`0312-0314` 表示3月12日至14日的汇总数据

**Q: 门店名称写得不一致能匹配吗？**  
A: 可以，工具会自动进行模糊匹配（相似度≥50%），比如：
- "朝阳门店" ↔ "北京朝阳门店"
- "海淀分店" ↔ "海淀店"

**Q: 日期格式支持哪些？**  
A: 支持多种格式：
- 标准格式：`2024-03-11`、`2024/03/11`
- 数字格式：`20250311`、`0311`（会自动补全年份为当年）
- 其他格式：`03/11`、`3-11` 等

**Q: 手续费和优惠怎么计算？**  
A: 使用**清分金额**（或清算金额）进行比对：
- 清分金额 = 交易金额 - 手续费 - 优惠金额
- 优先使用清分金额字段，如果没有则使用清算金额
- 都没有则使用：交易金额 - 手续费

**Q: "金额不符"是什么意思？**  
A: 表示流水汇总与结算单金额不一致：
- 结算金额 > 流水：可能缺少流水
- 结算金额 < 流水：流水有多余或结算单错误
