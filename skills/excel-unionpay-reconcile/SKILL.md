# 银联数据比对工具

## 功能概述

自动比对银联结算账单和银行流水，解决日常财务对账需求：

1. **数据预处理** - 自动识别日期格式、清洗门店名称、转换金额
2. **智能匹配** - 门店名称模糊匹配 + **按门店+日期分组汇总后金额比对**
3. **差异报告** - 输出未匹配流水组和结算单，便于核查

## 版本说明

- **V1 (unionpay_reconcile.py)**: 逐条流水匹配结算单
- **V2 (unionpay_reconcile_v2.py)**: 按门店+日期分组汇总后比对
- **V4 (unionpay_reconcile_v4.py)**: **推荐** - 增加金额反查，使用**清分金额**（扣除手续费和优惠后的实际到账金额）进行比对，解决有优惠场景下的匹配问题

## 使用方法

### V4 快速开始（最新推荐）

```bash
python scripts/unionpay_reconcile_v4.py 结算账单.xlsx 银行流水.xlsx
```

### V4 新特性

**1. 智能金额匹配**
- 优先使用**清分金额**（扣除手续费和优惠后的实际到账金额）
- 自动识别结算单中的优惠（钱包优惠、商户优惠等）

**2. 三层金额反查**
| 反查层级 | 说明 | 示例 |
|----------|------|------|
| 第一层 | 门店匹配+金额反查 | 宸海记济南：455.89+552.83=1008.72 |
| 第二层 | 纯金额反查 | 食满天天水：151.30=151.30 |
| 第三层 | 日期汇总反查 | 未莱嘉晟珠海：4512.39+1919.45=6431.84 |

**3. 输出报告结构**

生成的Excel报告包含11个工作表：

| Sheet | 内容 |
|-------|------|
| 比对结果 | 所有流水组的匹配情况 |
| 已匹配 | 完全匹配的记录 |
| 金额反查匹配 | 门店匹配+金额反查补足的记录 |
| 纯金额反查匹配 | 无门店名但金额匹配的记录 |
| **日期汇总反查匹配** | 按日期汇总无门店流水匹配的记录 |
| 金额不符 | 到账与结算金额不一致的记录 |
| 无法匹配 | 未找到对应结算单的记录 |
| 未匹配结算单 | 有结算单但无对应流水的记录 |
| 未匹配无门店流水 | 无门店名也未匹配的记录 |
| 所有流水明细 | 全部流水原始数据 |
| 汇总 | 统计信息 |

## V2 匹配逻辑（核心改进）

**按门店+日期分组汇总后比对**：

```
示例场景：
┌─────────────┬──────────┬──────────┐
│ 流水        │ 日期     │ 金额     │
├─────────────┼──────────┼──────────┤
│ 朝阳门店    │ 03/11    │ ¥2000    │
│ 朝阳门店    │ 03/11    │ ¥3000    │
└─────────────┴──────────┴──────────┘
              ↓ 汇总
        朝阳门店 03/11 = ¥5000
                    ↓ 比对
┌─────────────┬──────────┬──────────┐
│ 结算单      │ 日期     │ 金额     │
├─────────────┼──────────┼──────────┤
│ 朝阳门店    │ 0311     │ ¥5000    │ ← 匹配成功！
└─────────────┴──────────┴──────────┘
```

**匹配步骤（V4）：**
1. **门店映射** - 使用模糊匹配建立流水门店→结算单门店的映射
2. **分组汇总** - 按（映射后门店+日期）汇总流水和结算单
3. **金额比对** - 比较流水到账金额 vs 结算单**清分金额**
4. **第一层反查** - 对金额不符的，用无门店流水补足差额
5. **第二层反查** - 对无门店流水的，按金额匹配未匹配结算单
6. **第三层反查** - **按日期汇总剩余无门店流水，匹配同日期未匹配结算单**
7. **差额分析** - 显示最终差额帮助定位问题

## 输入数据要求

### 结算账单示例格式

| 门店 | 日期 | 交易金额 | 手续费 | 清分金额 |
|------|------|----------|--------|----------|
| 朝阳门店 | 2024-03-01 | 260.00 | 1.16 | 251.04 |
| 海淀分店 | 2024-03-01 | 152.00 | 0.70 | 151.30 |

**必需字段**：POS门店名称、结算日期、清分金额（或清算金额）

**说明**：清分金额 = 交易金额 - 手续费 - 优惠金额（如有）

### 银行流水示例格式

| 门店 | 日期 | 交易金额 | 手续费 |
|------|------|----------|--------|
| 北京朝阳门店 | 2024-03-01 | 5025.00 | 25.00 |
| 北京海淀分店 | 2024-03-01 | 3216.00 | 16.00 |

## 常见问题

**Q: "日期汇总反查"是什么意思？**  
A: 当多笔无门店名的流水（如支付宝/内卡入账）合计金额正好等于某门店某日的结算金额时，系统会自动按日期汇总这些流水，并匹配到对应门店。

示例：
```
无门店流水1：4512.39元（支付宝）
无门店流水2：1919.45元（内卡）
              ↓ 按0312日期汇总
              6431.84元
                    ↓ 匹配
未莱嘉晟珠海 0312 结算净额：6431.84元 ✓
```

**Q: V1、V2和V4有什么区别？**  
A: 三个版本的演进：
- **V1**：逐条流水匹配，适合一笔流水对应一笔结算
- **V2**：按门店+日期分组汇总后比对，适合多笔流水对应一笔结算
- **V4**：**推荐使用** - 在V2基础上增加三层金额反查，能处理无门店名、有优惠等复杂场景  
A: V1是逐条流水匹配，适合一笔流水对应一笔结算的场景；V2是按门店+日期分组汇总后比对，适合多笔流水汇总对应一笔结算的场景。**推荐使用V2**。

**Q: 门店名称写得不一致能匹配吗？**  
A: 可以，工具会自动进行模糊匹配（相似度≥60%），比如"朝阳门店"和"北京朝阳门店"会被识别为同一门店。

**Q: 日期格式支持哪些？**  
A: 支持多种格式：
- 标准格式：`2024-03-11`、`2024/03/11`
- 数字格式：`20250311`、`0311`（会自动补全年份为当年）
- 其他格式：`03/11`、`3-11` 等

**Q: 手续费和优惠怎么计算？**  
A: V4使用**清分金额**（或清算金额）进行比对，这是扣除手续费和优惠后的实际到账金额：
- 如果有清分金额字段：直接使用
- 如果有清算金额字段：使用清算金额
- 如果都没有：交易金额 - 手续费（兼容旧数据）

这样可以正确处理有优惠（如钱包优惠、商户优惠）的场景。

**Q: "金额不符"是什么意思？**  
A: 表示该门店该日期的流水净额汇总与结算单金额不一致：
- 结算金额 > 流水净额：可能缺少流水，差额就是漏掉的金额
- 结算金额 < 流水净额：流水有多余，或结算单金额有误

**Q: 发现未匹配记录怎么办？**  
A: 查看对应sheet分析原因：
- **未匹配流水组**：门店或日期对不上，检查门店名称映射是否正确
- **金额不符**：查看差额，差额就是漏掉的流水金额
- **未匹配结算单**：结算单有记录但无对应流水
